

# Verity — Full Quality Assurance Audit Report

## 1. Project Overview

**Verity** is an Australian 18+ anonymous speed-dating platform built on React 18, Vite, Tailwind CSS, and a Lovable Cloud backend. Users join scheduled "Drops" (45-second anonymous video calls via Agora RTC), make spark/pass decisions, and continue conversations through matched Sparks. The platform includes AI-powered moderation, a token/subscription economy via Stripe, and a transparency-first trust architecture.

**Target:** Australia-wide launch, primary domain `getverity.com.au`.

## 2. Critical Blockers Found (3 items — must fix before anything works)

### BLOCKER 1: Preview is broken — Config Error Screen showing

**Verified via browser screenshot:** The preview shows "Runtime configuration is incomplete" with `VITE_SUPABASE_URL` and `VITE_SUPABASE_PUBLISHABLE_KEY` missing.

**Root cause:** The `supabase/config.toml` contains `project_id = "lixgbpgemcpdbjzhjknf"` (an old external project), while the actual Lovable Cloud project is `itdzdyhdkbcxbqgukzis`. The `.env` file is auto-managed by Lovable Cloud based on the connected project. The `.env.production` also points to the wrong project.

**Fix:**
- Delete `.env.production` (it conflicts with Lovable Cloud's auto-managed `.env`)
- The `.env` is auto-generated by the platform — do not edit it
- `supabase/config.toml` must NOT be edited (per Lovable Cloud rules, it's auto-managed)
- The code in `runtimeEnv.ts` correctly reads from `import.meta.env` — the issue is the stale `.env.production` overriding values

### BLOCKER 2: Three critical database functions are missing

The code calls these RPCs but they do not exist in the database:

| Function | Called from | Impact |
|----------|-----------|--------|
| `claim_match_candidate(uuid, uuid)` | `find-match` edge function | **Matchmaking completely broken** — users cannot be matched |
| `submit_call_decision(uuid, spark_decision)` | `LiveCall.tsx` line 281 | **Spark/Pass decisions fail** — calls end with no outcome |
| `update_my_profile(text, text, text, text, text, text)` | `Profile.tsx` lines 35-36, 73 | **Profile name/avatar updates fail** |

**Root cause:** Migrations `20260228225500_3f9f0d2e-atomic-match-claim-rpc.sql` and `20260302043000_owner_mode_rls_hardening.sql` exist in code but were never applied to the Lovable Cloud database.

**Fix:** Apply these two migrations via the database migration tool.

### BLOCKER 3: `app_config` table does not exist

The `get-feature-flags` edge function queries `app_config.auth_policy`, but this table does not exist. The migration `20260302074500_app_config_feature_flags.sql` was never applied.

**Impact:** The `get-feature-flags` function returns a 500 error. However, `featureFlags.ts` has a fail-open default (`requirePhoneVerification: false`), so the app won't crash — it just can't enforce phone verification policy from the database.

**Fix:** Apply the `app_config` migration via the database migration tool.

## 3. High-Priority Issues (2 items)

### ISSUE 1: `.env.production` points to wrong project

File `.env.production` contains credentials for project `lixgbpgemcpdbjzhjknf` (an external project from a previous migration attempt). This file will override `.env` during production builds, pointing the frontend to the wrong backend.

**Fix:** Delete `.env.production` entirely. The Lovable Cloud `.env` is the source of truth.

### ISSUE 2: Database linter warning — Leaked Password Protection disabled

The linter flagged that leaked password protection is disabled for auth. This means users can sign up with passwords that have appeared in known data breaches.

**Fix:** Enable leaked password protection via the auth configuration.
